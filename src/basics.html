Text as strings of characters

But enough about numbers! We‚Äôre linguist, so let‚Äôs look at how we can represent text. The most basic way of representing text in Python is as one long string of characters. Strings can be created using quotes, both single and double work fine.

"Hello, world!"

"It's me, world!"

Though you can‚Äôt just put a single quote inside a single-quote-delimited string because, well, Python would think that‚Äôs where the string ends.

error messages that help you diagnose what went wrong, where the error
occurred and how you got there, so it's well worth spending time reading

Collections

Collections or containers are objects intended to contain other objects, so that you can conveniently manipulate them together. We saw above that almost all Python objects in some sense ‚Äúcontain‚Äù other objects via their attributes, but those attributes are somehow intimately tied to how any given object works. The whole point of collections is that they (mostly) don‚Äôt care what you put into them or what you take out; they‚Äôll just happily keep an eye on it for you and allow you to juggle and re-arrange the items in clever and succinct ways.

To motivate the need for collections, imagine that you want to store the individual tokens in a sentence, ‚ÄúLet it be.‚Äù. Without collections, you would have to use separate variables:

string1 = "Let"
string2 = "it"
string3 = "be"
string4 = "."

This gets really tedious really quickly. Instead, you can use a Python list:

strings = ["Let", "it", "be", "."]

Much better!

## Converting between collections

If you want to convert between the different types of collections, you
can mostly use built-in functions named after the target collection. For
instance, if you want to turn a set into a list:

```python
list({1, 2, 3, 1, 2, 3})

Or a list into a tuple:

tuple([1, 2, 3])

Or a tuple into a set:

set((1, 2, 3, 1))

With dictionaries, it‚Äôs slightly more complicated, as they don‚Äôt contain only values, but key‚Äìvalue pairs. When converting from a dictionary, you thus have to decide whether you want just the keys (the default, but you can also request it explicitly with the .keys() method), just the values, or key‚Äìvalue pairs ‚Äì so-called .items().

en2fr = {"cat": "chat", "dog": "chien"}
list(en2fr)

list(en2fr.keys())

tuple(en2fr.values())

set(en2fr.items())

Conversely, when converting to a dictionary, you have to provide a collection which can be interpreted as containing both keys and values, otherwise you can‚Äôt really build a dictionary out of it. One possible option is a list of 2-tuples.

dict([("a", "b"), ("c", "d")])

But it‚Äôs definitely not the only one. Try to understand and describe what‚Äôs going on in the next cell!

dict(["ab", "cd"])

The dict function also allows you to create a fresh dictionary in a way that may be slightly easier to type, with fewer curly braces and quotes, if your keys are strings which also happen to be valid identifiers (i.e., they could be used as variable names).

dict(cat="chat", dog="chien")

Strings are kind of the odd one out in this company because the str function doesn‚Äôt convert another collection to a string, at least not in the same sense the other functions we‚Äôve seen work. It returns a string representation of the collection, intended to suggest how you could create such a collection using literal syntax.

str([1, 2, 3])

str(en2fr)

In the other direction, the other collection functions split strings at character boundaries.

list("abracadabra")

set("abracadabra")

If you want to split anywhere else, you‚Äôll have to use the .split() method on strings. By default, it splits on whitespace, any amount and any kind of it.

"  foo\nbar  \n baz   qux  ".split()

But you can also tell it explicitly what string to use as a delimiter, and in that case, it‚Äôll follow your orders to the letter.

"  foo\nbar  \n baz   qux  ".split("\n")

Even creating empty strings if two designated delimiters immediately adjoin each other.

"  foo\nbar  \n baz   qux  ".split(" ")

The delimiter can consist of multiple characters.

"the cat sat on the mat".split("at")

Importing additional libraries

We‚Äôre about to dive into the magical world of conditionals and for-loops, but to make it more interesting, I thought we‚Äôd throw in some data and tools provided by the NLTK (which stands for Natural Language Toolkit) library. In order to do that however, we need to know how to import it, so bear with me for this short interlude.

In every Python session, some core functions and data types are available by default ‚Äì everything we‚Äôve seen so far is part of these so-called built-ins. In and of themselves, they‚Äôre already amazingly useful and allow you to do lots of stuff, but if everyone always had to start from these basic building blocks, programming would be repetitive and tedious. That‚Äôs why people build reusable pieces of code that can be imported into Python to extend its functionality. These are called libraries or packages or modules. Strictly speaking, each of these terms means slightly different things, but informally, they can be used interchangeably.

Import syntax in Python is simple and intuitive; it has a few basic variations which we‚Äôll presently go through.

import nltk

This imports the nltk module and creates an nltk variable which you can use to access the objects inside the module via attribute syntax. For instance, the word_tokenize() function splits text into words, or technically, tokens.

nltk.word_tokenize("Let it be.")

Notice how by default, Python tries to keep your objects and imported objects in separate namespaces, so that they don‚Äôt collide. If you happen to have previously defined a word_tokenize() function of your own, importing the nltk module won‚Äôt clobber it, because nltk‚Äôs word_tokenize() function is kept tucked away in the nltk namespace.

Of course, it may be the case that you actually have a previously created object named nltk that you don‚Äôt want to clobber. If so, then you can use renaming imports to pick the namespace yourself.

import nltk as ling

This imports the nltk module, but stores it in the variable / namespace ling instead of nltk.

ling.word_tokenize("Dig a pony.")

If you know you‚Äôre going to be using a specific object a lot and you don‚Äôt want to go to the trouble of typing the namespace prefix nltk. over and over again, then you can specifically request that it be added to your own namespace with the following syntax (notice that it‚Äôs customary to separate imports from regular code by at least one empty line):

from nltk import word_tokenize

word_tokenize("Two of us wearing raincoats.")

And of course, you can combine this with renaming if necessary.

from nltk import word_tokenize as tokenize

tokenize("I, me, mine.")

You can perform multiple imports per line by separating the different things to import with a comma.

import nltk, builtins
from builtins import len, set

If you want to import all objects defined in a module under the names given to them in that module, you can use star import syntax.

from builtins import *

At first, this might seem convenient and appealing. Those of you who know R might be intuitively drawn to this form because this is R‚Äôs default. Resist the urge. This variant makes it hard to track where objects came from, because a lot of names can hide under that *, not to mention if you do this with multiple modules. In time, you‚Äôll grow to appreciate Python‚Äôs more verbose but cleaner approach to namespaces, which makes it much easier to see at a glance where your variables came from.

Finally, the Python standard library contains many useful modules (the saying goes that Python comes with batteries included), but for many tasks, you‚Äôre likely to want to install additional packages. This can be done in a variety of different of ways, including a GUI manager if you‚Äôre using the Anaconda Python distribution, but the official Python package manager that should always be available is a command-line tool called pip.

This chapter is not about learning to use the command line, so just a quick crash course on pip. First, you need to figure out the name of the package you need. That can be done by searching the internet for keywords related to the functionality you want + Python, or by directly searching the Python Package Index.

If you get some sort of permission denied error, it‚Äôs because pip is trying to write into a system-wide library for all users which you don‚Äôt have write access to. In that case, try running pip install --user nltk instead.
